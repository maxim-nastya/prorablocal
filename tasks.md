# Дорожная карта развития "Прораб" (Roadmap)

Этот документ содержит список предстоящих задач для развития продукта. Текущая версия является **полностью локальной и работает в браузере**.

## ✅ Epic 1: Запуск MVP (Локальная версия) - ЗАВЕРШЕНО

**Цель: Создать полнофункциональное офлайн-приложение, работающее в браузере и хранящее данные локально.**

### 1.1. Backend и Инфраструктура
- [x] **~~Настройка сервера~~:** Не требуется. Приложение полностью клиентское.
- [x] **~~База данных~~:** Не требуется. Данные хранятся в `localStorage`.
- [x] **~~Разработка API (Django)~~:** Заменено локальным API-модулем (`api.ts`), работающим с `localStorage`.

### 1.2. Интеграция Frontend
- [x] **Аутентификация на Frontend:** Созданы страницы и логика для регистрации и входа локальных пользователей.
- [x] **Управление сессией:** Внедрено управление состоянием "сессии" пользователя.
- [x] **Перевод на API-запросы:** `localStorage` полностью заменен на асинхронные запросы к локальному API-модулю.
- [x] **Обработка состояний:** Реализованы индикаторы загрузки и обработка ошибок.

### 1.3. Монетизация (Симуляция)
- [x] **Логика подписки:** Реализован 3-дневный триал-период для новых пользователей.
- [x] **Страница оплаты:** Создана страница с симуляцией покупки подписки.
- [x] **Логика блокировки:** Реализована блокировка функционала по окончании триала/подписки.

### 1.4. Базовые Email-уведомления
- [ ] **(Требует Backend)** Интеграция с email-сервисом.
- [ ] **(Требует Backend)** Настройка транзакционных писем.

### 1.5. Админ-панель
- [ ] **(Требует Backend)** Настройка Django Admin для управления пользователями.

## 📈 Epic 2: Улучшения и расширение функционала (Post-MVP)

**Цель: Повысить ценность продукта на основе обратной связи от пользователей.**

- [ ] **(Требует Backend)** **Командный доступ:**
    - [ ] Возможность для бригадира приглашать сотрудников в свой аккаунт.
    - [ ] Настройка ролей и прав доступа.
- [ ] **Портал клиента 2.0:**
    - [ ] Добавить возможность клиенту оставлять комментарии к позициям сметы.
    - [x] Реализовать кнопку "Согласовать смету" для формального утверждения.

- [ ] **Улучшения по сметам:**
    - [x] Возможность создавать несколько смет для одного объекта.
    - [x] Добавление скидок (в % или фиксированной суммой) на всю смету.
    - [x] Генерация списка покупок из материалов сметы.
    - [ ] Управление (редактирование и удаление) позициями в личном справочнике.
- [ ] **Отчеты:**
    - [x] Генерация простых отчетов за период.
