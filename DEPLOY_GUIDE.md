# Руководство по запуску SaaS "Прораб" (Локальная версия)

**Версия:** 3.0
**Дата:** 29.08.2025

## 🎯 Введение

Этот документ — ваша дорожная карта для работы с **локальной версией** приложения "Прораб".

**Ключевое отличие:** Эта версия является полностью **клиентским (client-side) приложением**. Она работает автономно в вашем браузере и **не требует backend-сервера**. Все данные, которые вы создаете (проекты, сметы, пользователи), хранятся непосредственно в вашем браузере в `localStorage`.

---

## 🚀 Как запустить приложение

Для запуска приложения в режиме разработки, который идеально подходит для локального использования, вам понадобится Node.js.

### 1. Предварительные требования
-   **Установите Node.js:** Если у вас его нет, скачайте и установите LTS-версию с [официального сайта Node.js](https://nodejs.org/). Это даст вам инструмент `npm` (или `yarn`, `pnpm`) для управления зависимостями.

### 2. Установка зависимостей
Откройте терминал в корневой папке проекта и выполните команду. Она скачает и установит все необходимые для работы библиотеки (React, Vite и т.д.).
```bash
npm install
```

### 3. Запуск сервера для разработки
Выполните в терминале следующую команду:
```bash
npm run dev
```
После этого Vite запустит локальный веб-сервер, и вы увидите в терминале адрес, по которому доступно приложение (обычно `http://localhost:5173/`). Откройте эту ссылку в вашем браузере.

Теперь приложение полностью готово к работе!

---

## ⚙️ Как это работает

### Хранение данных
-   **localStorage:** Все ваши данные хранятся в специальном хранилище вашего браузера под названием `localStorage`.
-   **Приватность:** Данные никогда не покидают ваш компьютер.
-   **ВНИМАНИЕ:** Так как данные привязаны к конкретному браузеру, вы не сможете получить к ним доступ с другого устройства или из другого браузера на том же компьютере. **Очистка данных сайта или кеша браузера (cookies, site data) приведет к полному и безвозвратному удалению всех ваших проектов и настроек.**

### Офлайн-режим
Приложение является **Progressive Web App (PWA)**. После первой загрузки при наличии интернета, оно будет кешировать все необходимые файлы. Это означает, что вы сможете открывать и использовать его даже без подключения к сети.

## 📦 Сборка для "Продакшена" (Необязательно)

Если вы хотите разместить приложение на своем собственном веб-сервере или хостинге (например, Netlify, Vercel, или обычном Nginx), вам нужно создать оптимизированную "сборку".

1.  **Запустите сборку:**
    ```bash
    npm run build
    ```
2.  **Результат:** В корне проекта появится папка `dist/`. Содержимое этой папки — это готовый к развертыванию статический сайт.

Вы можете загрузить содержимое папки `dist/` на любой статический хостинг. В этом случае приложение также будет работать полностью локально в браузере у каждого пользователя, который его откроет, без необходимости в серверной части.
