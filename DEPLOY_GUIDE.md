# Руководство по запуску и развертыванию "Прораб" (Локальная версия)

**Версия:** 3.1
**Дата:** 30.08.2025

## 🎯 Введение

Этот документ — ваша дорожная карта для работы с **локальной версией** приложения "Прораб".

**Ключевое отличие:** Эта версия является полностью **клиентским (client-side) приложением**. Она работает автономно в вашем браузере и **не требует backend-сервера**. Все данные, которые вы создаете (проекты, сметы, пользователи), хранятся непосредственно в вашем браузере в `localStorage`.

---

## 🚀 Как запустить приложение локально

Для запуска приложения в режиме разработки, который идеально подходит для локального использования, вам понадобится Node.js.

### 1. Предварительные требования
-   **Установите Node.js:** Если у вас его нет, скачайте и установите LTS-версию с [официального сайта Node.js](https://nodejs.org/). Это даст вам инструмент `npm`.

### 2. Установка зависимостей
Откройте терминал в корневой папке проекта и выполните команду. Она скачает и установит все необходимые для работы библиотеки.
```bash
npm install
```

### 3. Запуск сервера для разработки
Выполните в терминале следующую команду:
```bash
npm run dev
```
После этого Vite запустит локальный веб-сервер, и вы увидите в терминале адрес, по которому доступно приложение (обычно `http://localhost:5173/`). Откройте эту ссылку в вашем браузере.

Теперь приложение полностью готово к работе!

## 🌐 Как развернуть на GitHub Pages

Вы можете легко опубликовать ваше приложение в интернете бесплатно с помощью GitHub Pages.

### Шаг 1: Настройте `package.json` (ОБЯЗАТЕЛЬНО!)

Откройте файл `package.json` и найдите строку `homepage`. Вам нужно заменить `YOUR_GITHUB_USERNAME` на ваше имя пользователя GitHub, а `prorab-frontend` — на название вашего репозитория, если оно другое.

**Пример:**
```json
// Если ваше имя пользователя 'ivanov' и репозиторий 'prorab-frontend'
"homepage": "https://ivanov.github.io/prorab-frontend",
```
**Это критически важный шаг! Без него стили и скрипты не подключатся.**

### Шаг 2: Запустите команду развертывания

Убедитесь, что все ваши изменения сохранены и отправлены в репозиторий на GitHub. Затем выполните в терминале одну команду:

```bash
npm run deploy
```

Эта команда сделает две вещи:
1.  **`predeploy`:** Соберет ваше приложение в оптимизированную для продакшена версию (в папку `dist`).
2.  **`deploy`:** Автоматически отправит содержимое папки `dist` в специальную ветку `gh-pages` вашего репозитория.

### Шаг 3: Включите GitHub Pages

1.  Перейдите в настройки вашего репозитория на GitHub (`Settings` -> `Pages`).
2.  В разделе "Build and deployment" выберите `Source` -> `Deploy from a branch`.
3.  Выберите ветку **`gh-pages`** и папку **`/ (root)`**. Нажмите `Save`.

Через несколько минут ваше приложение будет доступно по адресу, который вы указали в `package.json`.

## ⚙️ Как это работает

### Хранение данных
-   **localStorage:** Все ваши данные хранятся в специальном хранилище вашего браузера.
-   **Приватность:** Данные никогда не покидают ваш компьютер.
-   **ВНИМАНИЕ:** Очистка данных сайта или кеша браузера приведет к **полному и безвозвратному удалению** всех ваших проектов.

### Офлайн-режим
Приложение является **Progressive Web App (PWA)**. После первой загрузки оно будет работать даже без подключения к сети.
