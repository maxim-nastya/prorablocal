# Руководство по развертыванию "Прораб" на Vercel

**Версия:** 4.0
**Дата:** 31.08.2025

## 🎯 Введение

Этот документ описывает, как развернуть приложение "Прораб" на хостинг-платформе Vercel. Vercel идеально подходит для современных веб-приложений и предоставляет бесплатный тариф для персональных проектов.

**Ключевая информация:** Приложение является **клиентским (client-side)**, работает автономно в браузере и хранит все данные в `localStorage`. Для его работы не требуется бэкенд-сервер.

---

## 🌐 Процесс развертывания на Vercel

Развертывание на Vercel максимально автоматизировано.

### 1. Предварительные требования
-   **Аккаунт на GitHub (или GitLab/Bitbucket):** Ваш код должен находиться в Git-репозитории.
-   **Аккаунт на Vercel:** Зарегистрируйтесь на [vercel.com](https://vercel.com/), удобнее всего это сделать через ваш GitHub-аккаунт.

### 2. Импорт проекта в Vercel
1.  **Войдите в ваш Vercel-дашборд.**
2.  Нажмите кнопку **"Add New..." -> "Project"**.
3.  **Импортируйте Git-репозиторий:** Vercel покажет список ваших репозиториев. Найдите репозиторий с проектом "Прораб" и нажмите кнопку **"Import"**.

### 3. Настройка и развертывание
Vercel автоматически определит, что вы используете **Vite**, и выставит правильные настройки. Вам не нужно ничего менять.

-   **Framework Preset:** `Vite`
-   **Build Command:** `npm run build`
-   **Output Directory:** `dist`

Просто нажмите кнопку **"Deploy"**.

### 4. Готово!
Vercel автоматически установит зависимости, соберет проект и развернет его. Через 1-2 минуты ваше приложение будет доступно по сгенерированному домену (например, `prorab-frontend.vercel.app`).

---

## ⚙️ Как это работает

-   **Автоматические деплои:** Каждый раз, когда вы будете отправлять изменения в основную ветку вашего репозитория (например, `main`), Vercel будет автоматически запускать новый процесс сборки и обновлять ваше приложение.
-   **SPA-роутинг:** Файл `vercel.json` в корне проекта содержит правила, которые позволяют корректно работать внутренним ссылкам вашего приложения. Вам не нужно ничего дополнительно настраивать для этого.

### Хранение данных
-   **localStorage:** Все данные хранятся в специальном хранилище вашего браузера.
-   **Приватность:** Данные никогда не покидают компьютер пользователя.
-   **ВНИМАНИЕ:** Очистка данных сайта или кеша браузера приведет к **полному и безвозвратному удалению** всех проектов и данных.
